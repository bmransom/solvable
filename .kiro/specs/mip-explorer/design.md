# MIP Explorer - Design

## Overview

The MIP explorer combines three visualizations: (1) a 2D plot showing LP relaxation vs. integer points, (2) a branch-and-bound tree, and (3) a solve-time-vs-problem-size chart. These are separate Svelte components used in different lessons within Chapter 6.

## Components

### RelaxationComparison.svelte

**Purpose**: 2D plot showing feasible region, integer lattice, LP relaxation optimum, and integer optimum.

Built on top of the interactive plot with additional layers:
- Integer lattice: grid of dots at integer coordinates within viewport, colored by feasibility
- LP relaxation optimum marker: star shape, possibly fractional coordinates
- Integer optimum marker: diamond shape at integer coordinates
- "Nearest integer" marker: circle at the rounded LP solution, highlighted if not optimal
- Gap line: connects LP optimum to integer optimum with objective difference labeled

### BranchAndBoundTree.svelte

**Purpose**: Tree visualization of the B&B enumeration process.

Since `highs-js` doesn't expose B&B internals, this component implements a **pedagogical branch-and-bound** for small problems (up to ~15 binary variables) directly in TypeScript:

1. Solve the LP relaxation at the root
2. Pick the most fractional binary variable to branch on
3. Create two child nodes (variable = 0, variable = 1)
4. Solve each child's LP relaxation
5. Prune if infeasible, if bound is worse than incumbent, or if all integers are satisfied
6. Repeat

This custom implementation gives full control over pacing (step-by-step) and visualization (every node's data is available).

**Tree layout**: Horizontal tree using a simple recursive layout algorithm. Nodes are rounded rectangles with:
- Branching decision (e.g., "x3 = 0" / "x3 = 1")
- LP relaxation bound
- Status icon: green checkmark (integer feasible), red X (pruned), blue circle (unexplored)

**Incumbent / bound / gap display**: Three numbers prominently displayed above the tree, updating as the tree grows.

### SolveTimeChart.svelte

**Purpose**: Bar chart showing solve time vs. problem size.

- X-axis: number of binary variables (5, 8, 10, 12, 15, 18, 20, 25, 30)
- Y-axis: solve time (ms), log scale
- Each bar is generated by solving a parameterized knapsack problem via `highs-js`
- Bars fill in left-to-right as solves complete (async, using the Web Worker)
- A horizontal dashed line at 2 seconds marks "production latency budget"
- For bars above the line, show the incumbent/gap available at 2 seconds

### Pedagogical B&B Solver (branch-and-bound.ts)

**Purpose**: A simple, transparent B&B implementation for teaching.

Not a production solver — optimized for interpretability:
- Solves LP relaxations via `highs-js` (each node is a full LP solve with added variable bounds)
- Branching strategy: most fractional variable
- Node selection: best-first (highest bound for maximization)
- Returns the full tree structure with all node data
- Supports step-by-step mode: `step()` returns the next node to process

```typescript
interface BranchAndBoundNode {
  id: number;
  parent_id: number | null;
  branching_variable: string | null;
  branching_value: number;        // 0 or 1
  lp_relaxation_bound: number;
  is_integer_feasible: boolean;
  is_pruned: boolean;
  prune_reason: "infeasible" | "bounded" | "integer" | null;
  variable_values: Record<string, number>;
  children: [number, number] | null;  // [left_id, right_id]
}
```

## Testing Strategy

- **Pedagogical B&B**: Unit tests on small knapsack — verify optimal solution matches `highs-js` MIP solution, verify pruning logic
- **Relaxation comparison**: Verify LP optimum is a bound on integer optimum for test models
- **Solve time chart**: Verify solves complete and times are monotonically increasing (statistically)
